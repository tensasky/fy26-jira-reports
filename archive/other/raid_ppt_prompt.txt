培训内容大纲：

## 1. RAID 概述
- RAID = R风险 + A假设 + I问题 + D依赖关系
- 在 Jira 中管理 RAID：Work Type、Kanban Board

## 2. 四类 RAID 条目详解
- Risk（风险）：潜在事件，带影响和概率评分
- Action Item（行动项）：具体任务，有截止日期
- Issue（问题）：当前影响交付的活跃问题
- Decision（决策）：已做出的关键决策

## 3. 如何在 Jira 中创建 RAID
- 步骤：选择 Issue Type → 填写字段 → 创建
- 关键字段：Summary, Description, Assignee, Category, Mitigation Strategy

## 4. 风险与问题升级
- 三级升级模型：Team → Program → Enterprise
- Team Level：团队内解决，如 Sprint 范围风险
- Program Level：跨团队协调，如项目里程碑风险
- Enterprise Level：战略决策，如合规风险

## 5. RAID 层级与健康报告
- 层级：Story → Epic → Feature → Initiative → Goal
- 健康指标：风险暴露分数、问题优先级、逾期行动项
- 向上汇总逻辑：严重度 + 升级级别 + 趋势

## 6. 各层级收益
- 团队层：即时可见阻塞、明确缓解责任
- 项目群层：跨团队依赖意识、协调缓解策略
- 企业层：战略风险可见、资金优先级清晰

## 7. 最佳实践
- 每周审查 RAID 日志
- 清晰描述，明确影响范围和负责人
- 及时升级，不要等到最后一刻
- 持续改进，从问题中学习

## 8. RAID 条目模板
- 风险模板：描述、概率、影响、应对措施、负责人、日期
- 问题模板：描述、影响、升级路径、解决措施、负责人、日期
- 假设模板：描述、验证方法、风险、负责人
- 依赖模板：描述、依赖方、交付物、截止日期、负责人
