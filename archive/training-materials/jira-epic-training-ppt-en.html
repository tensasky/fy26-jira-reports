01 / STRUCTURE

Jira Hierarchy Structure

🏢 Initiative (Program)
↓
📁 Feature (Sub-project)
↓
📦 Epic (Work Package)
↓
📝 Story / Task
💡 Key Concept
When Intake enters the Delivery stage, it must be managed in Jira. Goal-Initiative-Feature has fixed parent-child relationships. Epic and below are managed in respective team Spaces.
02 / EPIC TYPES

Epic Types & Naming

1

Dev Delivery

Naming Convention:

Dev & SIT - Application Name

  • Each Epic is the smallest unit for deployment delivery
  • Application level, corresponding to PBN Level B
  • Epic ticket number must link to CAB change ticket
  • Multiple deployment teams may have multiple Epics distinguished by suffix
2

Project Management

Document Prefixes:

BRD PRD ISD FSD CSBD SRE Solution

Milestone Prefixes:

TUAT BUAT Penetration Test Cutover Plan Hypercare

03 / EPIC FIELDS

Epic Required Fields (13)

No Field Name Owner Instructions
1 Space System Jira Space name. Common names: Board, Space. Each Space has letter abbreviations like CNTIN, CNTD, CNDIN, etc.
2 Parent No PM Parent of current Ticket. Feature-Epic has fixed parent-child relationship. DO NOT modify this field arbitrarily.
3 Jira No System Composed of Space abbreviation + number, e.g., CNTIN-1234
4 Summary PM Refer to recommended prefix in Epic Breakdown + customized naming
5 Assignee PM Actual manager and operator. For Feature, this is the project PM. DO NOT modify arbitrarily.
03 / EPIC FIELDS (Cont.)

Epic Key Fields Details

📄 Description

Fixed Format:

  • 1. Risk log table
  • 2. Project document storage location

Confirmed document versions must be archived in the Description of the parent Feature.

📅 Date Fields

Start date - Planned start time

Due date - Planned end time

🔗 Linked Work Items

Covers all related Tickets beyond parent-child:

  • Related to: Represents Intake, managed by TPMO
  • Dependency: Represents dependencies, managed by PM

📊 Status & Status Trend

Status:

New → In Progress → Done

Status Trend:

Select based on actual project status, maintained by Program Manager

04 / STORY FIELDS

Story/Task Field Requirements

No Field Name Required Instructions
1 Space System Jira Space name. Common name: Board, Space.
2 Parent No Yes Parent of current Ticket. Epic-Story/Task has fixed parent-child relationship. Link in Parent field. DO NOT modify arbitrarily.
3 Summary Yes Custom naming based on actual task requirements
4 Assignee Yes Actual manager and operator. For Feature, this is the project PM.
5 Description Yes Fixed format: 1. Risk log table 2. Project document storage location
6 Start/Due date Yes Planned start and end time
7 Status Yes New (Not Started) → In Progress → Done (Business Live)

06 / RELATIONSHIPS

Parent-Child Relationships

Goal → Initiative → Feature

Goal
└─ Initiative

        └─ Epic
            └─ Story/Task

⚠️ DO NOT modify Parent fields arbitrarily!

📌 Space Distribution

China Technology Initiatives (CNTIN)

07 / BEST PRACTICES

Best Practices & FAQ

✅ Do's

  • Update Status in a timely manner
  • Fill in Start/End dates accurately
  • Include Risk log in Description
  • Link correct Parent tickets
  • Link CAB change ticket to Epic
  • Archive confirmed documents to Feature

❌ Don'ts

  • DO NOT modify Parent fields arbitrarily
  • DO NOT omit required fields
  • DO NOT create Tasks directly under Epic (create Story first)
  • DO NOT link unrelated tickets as Dependency
  • DO NOT delay updating Done status
